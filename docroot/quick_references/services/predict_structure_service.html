Protein Structure Prediction

Predicts 3D biomolecular structures (proteins, complexes, protein–DNA/RNA, protein–ligand) using Boltz-2, OpenFold 3, Chai-1, AlphaFold 2, or ESMFold.

The service unifies all five engines behind a single form. Choose Auto if you don't have a strong preference — the back end picks the tool that fits your inputs (e.g. ESMFold for a single protein with no MSA).

Confidence metrics (pLDDT, pTM, ipTM) and a normalized output layout are produced regardless of the engine, so results stay comparable.

Prediction Tool

The structure-prediction engine to run.

  • Auto — pick the engine that fits your inputs. Single protein, no MSA → ESMFold. Multi-chain or with ligands → Boltz-2 (requires MSA).
  • Boltz-2 — co-folds proteins with DNA/RNA/ligands. Requires an uploaded MSA.
  • OpenFold 3 — open re-implementation of AlphaFold 3 capabilities. Requires an MSA.
  • Chai-1 — another co-folding model with multimer and ligand support. Requires an MSA.
  • AlphaFold 2 — the classic monomer / multimer model. Builds its own MSA from local databases.
  • ESMFold — single-sequence protein prediction, no MSA. Fast; lower accuracy on rare folds.

Tool comparison and a decision tree: see Folding Tools Comparison in the project docs.

Biomolecular Inputs

Supply at least one FASTA file with Protein, DNA or RNA sequences. One file for sequence type, e.g. Protein. Files come from your workspace.

  • Protein — protein sequences. Multi-chain complexes: include multiple records in one file (max 26 sequences and 10,000 total residues per job). Boltz also accepts a YAML manifest for full feature support.
  • DNA — DNA sequences. Used as co-folding partners with proteins. Supported by Boltz-2, OpenFold 3, and Chai-1.
  • RNA — RNA sequences. Same engine support as DNA.

ESMFold and AlphaFold 2 ignore DNA/RNA inputs.

Ligands

Optional small-molecule ligands to co-fold with the proteins. Supported by Boltz-2 and Chai-1.

  • Ligands (CCD codes) — one or more PDB Chemical Component Dictionary codes (1–3 alphanumeric characters). One per line. Examples: ATP, NAD, NAG.
  • SMILES Strings — arbitrary small molecules expressed as SMILES. One per line. The form does live syntactic validation and surfaces the first invalid line.

Use CCD codes for standard cofactors and SMILES for novel compounds. Engines that don't support ligands ignore them.

Multiple Sequence Alignment

A pre-computed MSA file (.a3m, .sto, .pqt) used as input by some engines.

  • Required for Boltz-2, OpenFold 3, and Chai-1 — production BV-BRC does not run external MSA servers, so you must upload one.
  • Optional for Auto and ESMFold (ignored) and for AlphaFold 2 (which builds its own MSA from local databases).

To generate an MSA outside BV-BRC, use ColabFold's MMseqs2 server, JackHMMER against UniRef, or the standard AlphaFold preprocessing pipeline.

Output

Every submission creates a job with the name you give it. A workspace object named after the job is created inside the Output Folder and holds all job-related information — parameters, status, logs, and the prediction results themselves.

The full workspace path of the job object is shown in the Result location bar below the two fields: 

<Output Folder>/<Job Name>

Output Folder — an existing workspace folder. Defaults to your current workspace location.

Job Name — identifier for this run. Used as the workspace object name. Pick something short and descriptive (e.g. crambin-esmfold).

After the job finishes, open the workspace object to inspect the predicted structure, the confidence report, and the raw outputs. See the Output Results section of the Quick Reference for the full layout.
